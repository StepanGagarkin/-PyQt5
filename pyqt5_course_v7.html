Что такое PyQt5
Зачем использовать эту библиотеку
⌄

PyQt5 — это набор Python-привязок к библиотеке Qt5. Qt — одна из самых мощных платформ для создания GUI-приложений, которая работает на Windows, macOS и Linux.

Главное преимущество PyQt5: вы пишете на лаконичном Python, а получаете нативный интерфейс без сложных компиляторов.

В курсе мы создадим конвертер валют с реальными данными ЦБ РФ — сначала простой, затем расширим его до приложения с вкладками, графиками и журналом операций.

📌
PyQt5 даёт наилучшее соотношение «внешний вид / скорость разработки» для настольных приложений на Python.

Механизм сигналов и слотов
Главная концепция PyQt5
⌄

Сигнал — это событие, которое испускает виджет. Например, кнопка испускает сигнал clicked при нажатии.

Слот — это обычный Python-метод, который слушает сигнал и вызывается при его появлении.

Соединяются они методом connect() — это самая важная строка в любом PyQt5-приложении.

PYTHONПринцип сигнал → слот
# Общий синтаксис:
виджет.сигнал.connect(обработчик)

# Пример — кнопка:
self.convert_button.clicked.connect(self.convert_currency)
#  ↑ виджет         ↑ сигнал           ↑ метод-слот

# Другие сигналы:
self.line_edit.textChanged.connect(self.on_text_changed)
self.combo_box.currentIndexChanged.connect(self.on_select)
self.slider.valueChanged.connect(self.on_slide)
✅
Правило: одна строка сигнал.connect(метод) связывает событие интерфейса с логикой. Без этой строки кнопки и поля ввода ничего не делают.

Первое минимальное окно PyQt5
Hello World — структура любой программы
⌄

Любое PyQt5-приложение состоит из трёх обязательных элементов: QApplication, минимум одного виджета-окна и цикла событий app.exec_().

PYTHONhello_pyqt.py — первое окно
import sys
from PyQt5.QtWidgets import QApplication, QWidget, QLabel

class HelloWindow(QWidget):
    def __init__(self):
        super().__init__()          # ← ОБЯЗАТЕЛЬНО!
        self.setWindowTitle("Моё первое окно")
        self.setGeometry(300, 300, 400, 200)  # x,y,ширина,высота
        label = QLabel("Привет, PyQt5!", self)
        label.move(140, 80)

app = QApplication(sys.argv)   # ← создаём ПЕРВЫМ, до виджетов
window = HelloWindow()
window.show()
sys.exit(app.exec_())           # ← цикл событий

super().__init__() — инициализирует Qt-механизмы объекта. Без этого программа упадёт.

QApplication(sys.argv) — главный объект приложения, создаётся до любых виджетов.

app.exec_() — запускает бесконечный цикл обработки событий мыши, клавиатуры, таймеров.

Что такое QML и зачем он нужен
Декларативный подход против императивного
⌄

QML (Qt Modeling Language) — декларативный язык описания интерфейсов. Вместо того чтобы последовательно писать команды «создай кнопку», «задай текст», «добавь в макет», вы просто описываете, как должен выглядеть интерфейс в виде дерева объектов.

Ключевое слово — декларативный. Вы описываете что должно быть, а не как это создать. QML-движок сам берёт на себя создание и управление объектами.

В Способе 3 (чистый Python) вы пишете button = QPushButton("Кнопка") — это императивный стиль: шаг за шагом создаёте объект. В QML пишете Button { text: "Кнопка" } — это декларативный стиль: описываете результат. Внешне они дают одно и то же окно, но подход принципиально разный.

📌
QML — основа Qt для мобильных платформ. Все Qt-приложения для Android и iOS пишутся именно на QML. Освоив этот подход, вы сможете перенести конвертер на смартфон практически без изменений логики.
2
Разделение ответственности: что делает каждый файл
Почему логика и интерфейс живут отдельно
⌄

Главное преимущество архитектуры Python + QML — строгое разделение ответственности. Каждый файл знает только своё дело:

main.py — исключительно логика: HTTP-запросы к ЦБ РФ, парсинг XML, математические вычисления курсов. В этом файле нет ничего про кнопки, цвета или расположение элементов.

main.qml — исключительно внешний вид: расположение элементов, размеры окна, тексты меток, реакция на нажатия. В этом файле нет ни одной строки бизнес-логики.

Такое разделение называется паттерном MVVM (Model-View-ViewModel). На практике это означает: дизайнер может редактировать QML-файл, не зная Python. Разработчик может полностью переписать алгоритмы в Python, не трогая интерфейс.

Связь между файлами обеспечивается через три механизма: регистрацию Python-объекта в QML-контексте, сигналы (Python → QML) и слоты (QML → Python).

⚠️
Важное ограничение: Python не может напрямую взять QML-элемент и изменить его текст. Единственный легальный способ обновить интерфейс из Python — испустить сигнал через emit(). QML сам подписывается и реагирует. Это намеренно — нарушать разделение нельзя.

Импорты main.py: что и зачем подключаем
QObject, pyqtSignal, pyqtSlot, QQmlApplicationEngine
⌄

Импорты в main.py разделены на три смысловые группы. Разберём назначение каждой.

Стандартные библиотеки Python: sys нужен для передачи аргументов командной строки в QApplication и для передачи кода завершения в sys.exit(). requests выполняет HTTP-запрос к серверу ЦБ РФ. xml.etree.ElementTree парсит XML-ответ в дерево объектов Python.

Из PyQt5.QtCore импортируем три вещи: QObject — базовый класс Qt для объектов без визуального представления, pyqtSignal — фабрика сигналов (создаёт канал передачи данных из Python в QML), pyqtSlot — декоратор, помечающий метод как вызываемый из QML.

Из PyQt5.QtWidgets нужен только QApplication — главный объект любого PyQt5-приложения, управляет жизненным циклом и очередью событий.

Из PyQt5.QtQml нужен QQmlApplicationEngine — движок, который загружает .qml файл и создаёт из него живой интерфейс.

import xml.etree.ElementTree as ET

from PyQt5.QtCore import QObject, pyqtSlot, pyqtSignal
from PyQt5.QtWidgets import QApplication

Класс CurrencyConverter: QObject и pyqtSignal
Почему не QWidget, как объявляется сигнал
⌄

Почему наследуем от QObject, а не QWidget? QWidget — это визуальный элемент (окно, кнопка, поле). Наш класс не рисуется на экране — он содержит только логику. QObject даёт систему сигналов и слотов без лишних GUI-накладных расходов.

Строка resultChanged = pyqtSignal(str) объявляет сигнал на уровне класса (не экземпляра). pyqtSignal(str) означает: сигнал будет нести одно строковое значение. Имя resultChanged выбрано произвольно, но оно важно: QML будет ожидать функцию-обработчик с именем onResultChanged (правило: on + имя сигнала с заглавной буквы).

Конструктор __init__: super().__init__() обязательно вызывается первым — Qt инициализирует внутренние механизмы объекта. Затем создаём пустой словарь self.rates, который будет хранить курсы вида {"USD": 92.5, "EUR": 100.2}, и сразу вызываем load_rates() — курсы загружаются ещё до открытия окна.

PYTHONОбъявление класса и сигнала
class CurrencyConverter(QObject):

    resultChanged = pyqtSignal(str)  # сигнал несёт одну строку

    def __init__(self):
        super().__init__()
        self.rates = {}
        self.load_rates()
💡
Типы данных в pyqtSignal могут быть разными: pyqtSignal(str), pyqtSignal(int), pyqtSignal(float), pyqtSignal(bool), pyqtSignal(list). Тип определяет, что будет передано обработчику в QML при вызове emit().

Метод load_rates: парсинг XML от ЦБ РФ
requests.get, ElementTree, Nominal — зачем нужно деление
⌄

requests.get(url) выполняет синхронный HTTP-запрос — приложение ждёт ответа сервера, прежде чем идти дальше. Результат r.content — байтовая строка XML.

ET.fromstring(r.content) парсит байты в дерево объектов Python. root — это корневой тег <ValCurs>, родитель всех валют.

root.findall("Valute") возвращает список всех дочерних тегов <Valute>. В ответе ЦБ их обычно 35–40 штук — по одному на каждую валюту.

Для каждой валюты извлекаем три поля:

• CharCode — буквенный код: "USD", "EUR", "JPY". Станет ключом словаря.

• Value — курс в формате "92,5000" (запятая вместо точки, российский стандарт). Метод .replace(",",".") делает его читаемым для Python, float() — числом.

• Nominal — за сколько единиц дан курс. Для USD и EUR это 1, но для JPY — 100, для KZT — 100. Без деления на Nominal курс иены был бы завышен в 100 раз.

Итоговая формула: self.rates[code] = val / nom даёт цену ровно одной единицы каждой валюты в рублях.

Блок except: вместо аварийного завершения программы при ошибке сети — испускаем сигнал с текстом ошибки. QML отобразит его в той же метке результата.

PYTHONМетод load_rates
    def load_rates(self):
        try:
            r = requests.get("https://www.cbr.ru/scripts/XML_daily.asp")
            root = ET.fromstring(r.content)
            for v in root.findall("Valute"):
                code = v.find("CharCode").text
                val = float(v.find("Value").text.replace(",", "."))
                nom = int(v.find("Nominal").text)
                self.rates[code] = val / nom
        except Exception as e:
            self.resultChanged.emit(f"Ошибка загрузки: {e}")
XMLСтруктура ответа ЦБ РФ

    <CharCode>USD</CharCode>   <!-- ключ словаря -->
    <Nominal>1</Nominal>

  </Valute>
  <Valute ID="R01820">
    <CharCode>JPY</CharCode>

Итоговые значения словаря: rates["USD"] = 92.5 / 1 = 92.50  ·  rates["JPY"] = 62.3 / 100 = 0.623  ·  rates["EUR"] = 100.2 / 1 = 100.20

Метод convert_currency: @pyqtSlot и emit
Декоратор слота, обработка ввода, испускание сигнала
⌄

Декоратор @pyqtSlot(str, str) обязателен для методов, которые вызываются из QML. Аргументы в скобках — это типы параметров метода: оба аргумента будут строками. Без этого декоратора QML не сможет найти метод по имени.

Параметры метода: amount_text — строка из TextField в QML (например, "100" или "abc"). currency_text — строка из ComboBox (например, "USD (Доллар)"). Оба всегда приходят как строки, даже если пользователь ввёл число.

Блок try/except для float(): если пользователь ввёл нечисловой текст ("abc", "", "--"), float() выбросит ValueError. Вместо аварийной остановки — испускаем сигнал с сообщением об ошибке и прерываем метод через return.

Извлечение кода валюты: строка "USD (Доллар)" → .split() даёт список ["USD", "(Доллар)"] → [0] берёт первый элемент "USD". Именно под этим ключом хранится курс в словаре self.rates.

self.rates.get(code, 0) безопаснее, чем self.rates[code]: если ключ не найден — вернёт 0 вместо KeyError.

Вызов emit() "отправляет" сигнал с форматированной строкой. Форматирование :.2f задаёт два знака после запятой (9250.00, а не 9250.0). QML получит эту строку в обработчике onResultChanged.

PYTHONМетод convert_currency
    @pyqtSlot(str, str)
    def convert_currency(self, amount_text, currency_text):
        try:
            amount = float(amount_text)
        except ValueError:
            self.resultChanged.emit("Введите корректное число")
            return
        code = currency_text.split()[0]
        result = amount * self.rates.get(code, 0)
        self.resultChanged.emit(f"Результат: {result:.2f} RUB")

Точка входа: QApplication, Engine, setContextProperty
Порядок инициализации и связь Python-объекта с QML
⌄

Блок if __name__ == "__main__" гарантирует: код выполняется только при прямом запуске файла (python main.py), но не при импорте как модуля.

QApplication(sys.argv) — главный объект приложения. Создаётся до любых виджетов и движков. sys.argv содержит аргументы командной строки; Qt использует их для системных настроек (тема, масштаб, DPI).

QQmlApplicationEngine() — создаём движок. Он умеет загружать .qml файлы и создавать из них живые объекты интерфейса.

CurrencyConverter() — создаём экземпляр. В конструкторе автоматически вызовется load_rates() — курсы загрузятся ещё до того, как откроется окно.

engine.rootContext().setContextProperty("backend", converter) — ключевая строка. rootContext() — корневое пространство имён QML-движка. setContextProperty("backend", converter) регистрирует наш Python-объект под именем "backend". После этого из любого места QML-файла можно писать backend.convert_currency(...) или подписываться на backend.resultChanged. Эту строку нужно вызвать до engine.load() — иначе QML-файл загрузится раньше, чем объект появится в контексте.

engine.load("main.qml") — загружает файл, создаёт все QML-объекты, связывает имя "backend" с нашим converter и отображает окно.

app.exec_() запускает цикл событий — программа ждёт здесь нажатий, ввода, таймеров. sys.exit() передаёт код завершения операционной системе (0 = успех).

PYTHONТочка входа main.py
if __name__ == "__main__":
    app = QApplication(sys.argv)
    engine = QQmlApplicationEngine()
    converter = CurrencyConverter()
    engine.rootContext().setContextProperty("backend", converter)
    engine.load("main.qml")
    sys.exit(app.exec_())
⚠️
Порядок строк критичен: сначала setContextProperty, только потом engine.load(). Если поменять местами — QML-файл загрузится до регистрации объекта, и backend будет undefined.

Файл main.qml: структура и элементы интерфейса
ApplicationWindow, Column, Label, TextField, ComboBox, Button
⌄

QML-файл строится как дерево вложенных объектов. Каждый объект — это блок с именем типа и фигурными скобками, внутри которых задаются свойства и дочерние объекты.

Импорты QML — аналог import в Python. QtQuick 2.15 даёт базовые типы (Item, Rectangle, Text). QtQuick.Controls 2.15 даёт виджеты управления (Button, TextField, ComboBox, Label).

ApplicationWindow — корневой элемент, создаёт системное окно. Свойство visible: true обязательно: без него окно создаётся, но не отображается. width и height — начальный размер в пикселях. title — текст заголовка.

Column — контейнер, располагающий дочерние элементы вертикально сверху вниз. anchors.fill: parent растягивает контейнер на всю площадь родителя (окна). anchors.margins: 10 добавляет отступ 10px от всех четырёх сторон. spacing: 8 — 8px пустого пространства между каждым дочерним элементом.

Label — нередактируемая текстовая метка, аналог QLabel в Python. Свойство text задаёт содержимое.

TextField — однострочное поле ввода, аналог QLineEdit. id: amountInput — уникальный идентификатор внутри QML-файла, позволяет другим элементам ссылаться на этот объект. placeholderText — подсказка, отображаемая пока поле пустое. Текущее содержимое читается как amountInput.text.

ComboBox — выпадающий список. id: currencyBox нужен для чтения currencyBox.currentText в обработчике кнопки. Свойство model принимает массив строк в квадратных скобках.

Button — нажимаемая кнопка. onClicked — обработчик события нажатия. Внутри него вызывается backend.convert_currency(...) — метод Python-объекта, зарегистрированного в контексте движка. Обращение к id других элементов (amountInput.text, currencyBox.currentText) работает из любого места того же QML-файла.

Блок Connections: подписка QML на Python-сигнал
Как данные идут от Python к интерфейсу — шаг за шагом
⌄

Connections — специальный QML-объект для подписки на сигналы внешнего объекта. Он не визуальный, не занимает место в интерфейсе, поэтому размещается на уровне ApplicationWindow, а не внутри Column.

Свойство target: backend указывает: слушаем сигналы именно этого объекта. backend — имя, под которым Python-объект был зарегистрирован через setContextProperty.

Имя функции-обработчика строится по строгому правилу: on + имя сигнала с первой заглавной буквой. Сигнал называется resultChanged → функция называется onResultChanged. Регистр важен: onresultchanged работать не будет.

Параметр result — строка, переданная через emit() на стороне Python. Внутри функции присваиваем её свойству text метки resultLabel. resultLabel — это id последнего Label в Column.

Полный путь данных при нажатии кнопки:

1. Пользователь нажимает Button → QML выполняет onClicked

2. QML вызывает backend.convert_currency(amountInput.text, currencyBox.currentText)

3. Qt находит Python-метод с @pyqtSlot(str, str) и вызывает его

4. Python выполняет расчёт: конвертирует строку в число, ищет курс в словаре

5. Python вызывает self.resultChanged.emit("Результат: 9250.00 RUB")

6. Qt помещает сигнал в очередь событий и находит подписчиков в QML

7. QML вызывает onResultChanged("Результат: 9250.00 RUB")

8. Устанавливается resultLabel.text = "Результат: 9250.00 RUB" — пользователь видит ответ

⌄

В сгенерированный файл добавляем свою логику после вызова retranslateUi(Form) — это безопасная зона, которая не перезапишется при повторной конвертации.

Три строки нашего кода в setupUi:

• self.rates = {} — пустой словарь курсов

• self.load_rates() — загружаем курсы при создании

• self.pushButton.clicked.connect(self.convert_currency) — подключаем кнопку к обработчику

Методы load_rates() и convert_currency() добавляем ниже как методы класса Ui_Form.

PYTHONconverter.py — полный код с логикой
from PyQt5 import QtCore, QtGui, QtWidgets
import requests, xml.etree.ElementTree as ET

class Ui_Form(object):
    def setupUi(self, Form):
        # --- СГЕНЕРИРОВАННЫЙ КОД ---
        Form.resize(248, 200)
        self.verticalLayout = QtWidgets.QVBoxLayout(Form)
        self.labelAmount = QtWidgets.QLabel(Form)
        self.verticalLayout.addWidget(self.labelAmount)
        self.lineEditAmount = QtWidgets.QLineEdit(Form)
        self.verticalLayout.addWidget(self.lineEditAmount)
        self.comboBoxCurrency = QtWidgets.QComboBox(Form)
        self.verticalLayout.addWidget(self.comboBoxCurrency)
        self.pushButton = QtWidgets.QPushButton(Form)
        self.verticalLayout.addWidget(self.pushButton)
        self.labelResult = QtWidgets.QLabel(Form)
        self.verticalLayout.addWidget(self.labelResult)
        self.retranslateUi(Form)
        # --- НАШ КОД (добавляем после retranslateUi) ---
        self.rates = {}
        self.load_rates()
        self.pushButton.clicked.connect(self.convert_currency)

    def retranslateUi(self, Form):
        _translate = QtCore.QCoreApplication.translate
        Form.setWindowTitle(_translate("Form", "Конвертер"))
        self.labelAmount.setText(_translate("Form", "Введите сумму:"))
        self.comboBoxCurrency.setItemText(0, _translate("Form", "USD (Доллар)"))
        self.comboBoxCurrency.setItemText(1, _translate("Form", "EUR (Евро)"))
        self.pushButton.setText(_translate("Form", "Рассчитать"))
        self.labelResult.setText(_translate("Form", "Результат:"))

    # --- ДОБАВЛЕННЫЕ МЕТОДЫ ---
    def load_rates(self):
        try:
            r = requests.get("https://www.cbr.ru/scripts/XML_daily.asp")
            root = ET.fromstring(r.content)
            for v in root.findall("Valute"):
                self.rates[v.find("CharCode").text] = (
                    float(v.find("Value").text.replace(",","."))
                    / int(v.find("Nominal").text))
        except: self.labelResult.setText("Ошибка загрузки")

    def convert_currency(self):
        try: amount = float(self.lineEditAmount.text().replace(",","."))
        except: self.labelResult.setText("Введите число"); return
        code = self.comboBoxCurrency.currentText().split()[0]
        result = amount * self.rates.get(code, 0)
        self.labelResult.setText(f"Результат: {result:.2f} RUB")

if __name__ == "__main__":
    import sys
    app = QtWidgets.QApplication(sys.argv)
    Form = QtWidgets.QWidget()
    ui = Ui_Form()
    ui.setupUi(Form)
    Form.show()
    sys.exit(app.exec_())

Принцип классического подхода
Чем отличается от QML и Qt Designer
⌄

В Способе 3 весь интерфейс создаётся программно на чистом Python — никаких внешних файлов (.qml, .ui), никаких кодогенераторов. Каждый виджет создаётся вручную, каждое соединение прописывается явно.

Это самый прямолинейный и прозрачный подход: вы точно знаете, какой объект существует в программе, потому что сами его создали. Нет скрытых генераций, нет «магии» движка.

Сравнение с другими способами:

• Способ 1 (QML) — интерфейс в отдельном файле .qml, Python только логика. Два файла, строгое разделение.

• Способ 2 (Qt Designer) — интерфейс рисуется мышью, затем генерируется в Python. Быстро, но теряется контроль над кодом.

• Способ 3 (классический) — всё в одном Python-файле. Медленнее писать, но каждая строка понятна и находится под контролем.

Именно Способ 3 даёт глубокое понимание того, как работают виджеты. После его освоения Способы 1 и 2 становятся очевидными.

Почему класс = окно, роль super().__init__()
⌄

Строка class CurrencyConverter(QWidget) — объявляем класс, который наследует от QWidget. Это означает: наш класс сам является окном. Он наследует все методы QWidget: show(), hide(), setWindowTitle(), resize() и сотни других.

В Способе 2 (Qt Designer) был отдельный QWidget-контейнер, которому передавали класс Ui_Form. Здесь этого нет — наш класс сам является и контейнером, и логикой одновременно.

Строка super().__init__() вызывает конструктор родительского класса QWidget. Это обязательно: Qt инициализирует внутренние C++-объекты, систему событий, механизм отрисовки. Без этого вызова программа упадёт с RuntimeError при первом же обращении к любому методу виджета.

Правило простое: первая строка любого __init__ в PyQt5 — всегда super().__init__().

        super().__init__()
⚠️
Типичная ошибка: пропустить super().__init__() или поставить его не первой строкой. Результат — RuntimeError: super().__init__() was never called в момент вызова любого метода Qt.

QVBoxLayout: менеджер компоновки
Как виджеты выстраиваются в столбец автоматически
⌄

QVBoxLayout (Vertical Box Layout) — менеджер компоновки, который автоматически располагает добавленные в него виджеты вертикально сверху вниз. Без менеджера компоновки виджеты накладывались бы друг на друга в левом верхнем углу окна.

Принцип работы состоит из трёх шагов, которые нужно соблюдать строго в этом порядке:

Шаг А: создать объект макета — layout = QVBoxLayout(). В этот момент макет пустой, ни с чем не связан.

Шаг Б: добавлять виджеты через layout.addWidget(виджет). Порядок вызовов определяет порядок виджетов сверху вниз — первый добавленный окажется выше, последний ниже.

Шаг В: применить макет к окну — self.setLayout(layout). Без этого вызова макет создан, виджеты добавлены, но ничего не отображается — макет не знает, к какому окну он относится.

Дополнительные методы: addSpacing(px) добавляет пустое пространство нужной высоты между виджетами. addStretch() добавляет резиновый распорщик — растягивающееся пустое пространство, которое выравнивает оставшиеся виджеты.

QPushButton и сигнал clicked
Создание кнопки, механизм connect, что такое слот
⌄

QPushButton — стандартная нажимаемая кнопка. Создаётся с текстом, который будет отображаться на ней: QPushButton("Перевести в рубли").

Сама по себе кнопка ничего не делает при нажатии — нужно явно указать, что должно произойти. Это делается через механизм сигналов и слотов.

Строка self.btn.clicked.connect(self.convert_currency) — ключевая строка для любой кнопки. Разберём её по частям:

• self.btn — наш объект кнопки

• .clicked — сигнал кнопки, который испускается при нажатии

• .connect(...) — метод, который подписывает обработчик на этот сигнал

• self.convert_currency — ссылка на метод-обработчик (без скобок — мы передаём сам метод, а не вызываем его)

После этой строки Qt запомнил: «когда кнопка нажата — вызвать self.convert_currency()». Один сигнал может быть подключён к нескольким обработчикам — все они будут вызваны при нажатии.

Важно: передаётся self.convert_currency без скобок, не self.convert_currency(). Со скобками — это вызов метода прямо сейчас. Без скобок — передача ссылки на метод для будущего вызова.

QMessageBox и метод convert_currency
Диалоги ошибок, чтение виджетов, setText результата
⌄

QMessageBox — стандартное диалоговое окно операционной системы. Блокирует главное окно до тех пор, пока пользователь не закроет диалог. Вызывается как статический метод — экземпляр создавать не нужно.

Три варианта в порядке серьёзности:

• QMessageBox.critical(self, "Заголовок", "Текст") — красная иконка ✗, для критических ошибок (нет сети, файл не найден)

• QMessageBox.warning(self, "Заголовок", "Текст") — жёлтая иконка ⚠, для предупреждений (неверный ввод пользователя)

• QMessageBox.information(self, "Заголовок", "Текст") — синяя иконка ℹ, для информационных сообщений

Параметр self (первый аргумент) — родительское окно. Qt центрирует диалог относительно этого окна.

В методе convert_currency происходит следующее. Метод self.amount_input.text() читает текущее содержимое поля ввода как строку. Оборачиваем в float() внутри try — если пользователь ввёл нечисловой текст, float() выбросит ValueError, мы поймаем его и покажем предупреждение.

Метод self.currency_box.currentText() возвращает строку вида "USD (Доллар)". Вызов .split()[0] разбивает строку по пробелам и берёт первый элемент — "USD". Это ключ для словаря self.rates.

После вычисления результата вызываем self.result_label.setText(f"Результат: {result:.2f} RUB"). Форматирование :.2f даёт два знака после запятой. Метод setText() обновляет текст метки — пользователь сразу видит изменение на экране.

Точка входа в классическом PyQt5-приложении всегда выглядит одинаково — три строки после блока if __name__ == "__main__".

QApplication(sys.argv) создаётся первым, до любых виджетов. Именно он инициализирует графическую подсистему операционной системы — без этого объекта создать окно невозможно.

        if self.amount_max_input.text():
            data=[x for x in data if float(x[amount])<=float(self.amount_max_input.text())]

            root = ET.fromstring(response.content)
            for valute in root.findall(Valute):
                char=valute.find(CharCode).text
                val=float(valute.find(Value).text.replace(,,.))
                nom=int(valute.find(Nominal).text)
                self.rates[char]=val/nom
            self.graph_tab.update_graph()
        except: pass
